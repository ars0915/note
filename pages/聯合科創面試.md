		- #### 2. Streaming Protocols
			- **可能的問題：**
				- 「RTMP, HLS, DASH 的差異和適用場景？」
				- 「WebRTC vs SRT 的差異？」
				- 「如何降低串流延遲？」
				- 「如何做 adaptive bitrate streaming？」
			- **準備要點：**
			  ```
			  Protocol 比較：
			  - RTMP: 低延遲，但過時，主要用在 ingest
			  - HLS: 高延遲（6-30s），但相容性好
			  - DASH: 類似 HLS，更現代
			  - WebRTC: 超低延遲（<1s），但複雜
			  - SRT: 低延遲，可靠，適合長距離
			  
			  延遲優化：
			  - 減少 buffer size
			  - 使用更快的 codec
			  - 優化 network path
			  - 使用 WebRTC/SRT 而非 HLS
			  ```
